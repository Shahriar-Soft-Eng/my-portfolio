The Refactoring Glitch

In Unity game development, field names directly map to asset serialization files (.prefab, .unity, .asset). When a programmer renames a serialized C# variable (e.g. changing scoreLimit to maxScore):

  • Unity loses the matching serialized keys, silently resetting data states.
  • Enemy attributes, structural links, and game configurations corrupt.
  • Dev teams lose hundreds of active hours re-inputting inspector values.

The Serialized Shield

Serialized Shield listens to file change streams. When a variable name edit is executed, the AST Synthesizer analyzes standard fields, automatically injecting Unity's native recovery attribute:

[FormerlySerializedAs("scoreLimit")]
private int maxScore;

This preserves serialization mappings perfectly. Re-imported scripts load existing data states with zero asset breaks.

// Detect rename operation in C# Syntax tree and inject FormerlySerializedAs
export function registerRenameHook(context: vscode.ExtensionContext) {
  vscode.workspace.onDidChangeTextDocument((event) => {
    const changes = event.contentChanges;
    const document = event.document;

    if (document.languageId !== 'csharp') return;

    for (const change of changes) {
      const match = detectVariableRename(change.text, document);
      if (match) {
        injectSerializationShield(document, match.line, match.oldName);
      }
    }
  });
}
